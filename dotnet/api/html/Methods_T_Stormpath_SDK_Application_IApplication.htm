AddAccountStoreAsync(String, CancellationToken)
Adds a new IAccountStore to this Application. The given string can either be an href or a name of a IDirectory or IGroup belonging to the current ITenant.

If the provided value is an href, this method will get the proper Resource and add it as a new AccountStore in this Application without much effort. However, if the provided value is not an href, it will be considered as a name. In this case, this method will search for both a Directory and a Group whose names equal the provided hrefOrName. If only one resource exists (either a Directory or a Group), then it will be added as a new AccountStore in this Application. However, if there are two resources (a Directory and a Group) matching that name, a ResourceException will be thrown.

Note: When using names this method is not efficient as it will search for both Directories and Groups within this Tenant for a matching name. In order to do so, some looping takes place at the client side: groups exist within directories, therefore we need to loop through every existing directory in order to find the required Group. In contrast, providing the Group's href is much more efficient as no actual search operation needs to be carried out.

AddAccountStoreAsync(IAccountStore, CancellationToken)
Adds a new IAccountStore to this Application and appends the resulting IAccountStoreMapping to the end of the Application's AccountStoreMapping list.

If you need to control the order of the added AccountStore, use the CreateAccountStoreMappingAsync(IAccountStoreMapping, CancellationToken) method.

GetDefaultGroupStoreAsync
Gets the IAccountStore used to persist new groups created by the application, or if no account store has been designated.

GetGroups
Gets a queryable list of all groups accessible to this application. It will not only return any group associated directly as an IAccountStore but also every group that exists inside every directory associated as an account store.

NewIdSiteAsyncCallbackHandler
Creates a new IIdSiteAsyncCallbackHandler used to handle HTTP replies from your ID Site to your application's callbackUri, as described in the NewIdSiteUrlBuilder method.
NewIdSiteUrlBuilder
Creates a new IIdSiteUrlBuilder that allows you to build a URL you can use to redirect your application users to a hosted login/registration/forgot-password site - what Stormpath calls an 'Identity Site' (or 'ID Site' for short) - for performing common user identity functionality. When the user is done (logging in, registering, etc), they will be redirected back to a callbackUri of your choice.
ResetPasswordAsync
Verifies the password reset token (received in the user's email) and immediately changes the password in the same request, if the token is valid.

Once the token has been successfully used, it is immediately invalidated and can't be used again. If you need to change the password again, you will previously need to execute SendPasswordResetEmailAsync(String, CancellationToken) again in order to obtain a new password reset token.

SendPasswordResetEmailAsync(String, CancellationToken)
Sends a password reset email for the specified account email address. The email will contain a password reset link that the user can click or copy into their browser address bar.
SendPasswordResetEmailAsync(String, IAccountStore, CancellationToken)
Sends a password reset email to an account in the specified IAccountStore matching the specified email. If the email does not match an account in the specified IAccountStore, a ResourceException will be thrown. If you are unsure of which of the application's mapped account stores might contain the account, use the more general SendPasswordResetEmailAsync(String, CancellationToken) method instead. The email will contain a password reset link that the user can click or copy into their browser address bar.
SendPasswordResetEmailAsync(String, String, CancellationToken)
Sends a password reset email to an account matching the specified email in the AccountStore or Organization matching the specified hrefOrNameKey. If the email does not match an account in the specified AccountStore or Organization, a ResourceException will be thrown. If you are unsure of which of the application's mapped account stores might contain the account, use the more general SendPasswordResetEmailAsync(String, CancellationToken) method instead. The email will contain a password reset link that the user can click or copy into their browser address bar.
SendVerificationEmailAsync(String, CancellationToken)
Triggers the delivery of a new verification email for the specified account.

This method is useful in scenarios where the Account Registration and Verification workflow is enabled. If the welcome email has not been received by a newly registered account, then the user will not be able to login until the account is verified.

This method re-sends the verification email and allows the user to verify the account.

SendVerificationEmailAsync(ActionEmailVerificationRequestBuilder, CancellationToken)
Triggers the delivery of a new verification email for the specified account.

This method is useful in scenarios where the Account Registration and Verification workflow is enabled. If the welcome email has not been received by a newly registered account, then the user will not be able to login until the account is verified.

This method re-sends the verification email and allows the user to verify the account.

The IEmailVerificationRequest must contain the username or email identifying the account.

SetDefaultAccountStoreAsync
Sets the IAccountStore (either a IGroup or a IDirectory) used to persist new accounts created by the Application.

Because an Application is not an IAccountStore itself, it delegates to a Group or Directory when creating accounts; this method sets the IAccountStore to which the Application delegates new account persistence.

SetDefaultGroupStoreAsync
Sets the IAccountStore (a IDirectory) used to persist new groups created by the Application.

Stormpath's current REST API requires this to be a Directory. However, this could be a Group in the future, so do not assume it is always a Directory if you want your code to function properly if/when this support is added.

Because an Application is not an IAccountStore itself, it delegates to a Directory when creating groups; this method sets the IAccountStore to which the Application delegates new group persistence.
