AddAccountStoreTMapping(IAccountStoreContainerTMapping, IAccountStore)
Synchronously adds a new IAccountStore to the Application or Organization and appends the resulting IAccountStoreMapping to the end of the AccountStoreMapping list.

If you need to control the order of the added AccountStore, use the CreateAccountStoreMappingTMapping(IAccountStoreContainerTMapping, TMapping) method.

AddAccountStoreTMapping(IAccountStoreContainerTMapping, String)
Synchronously adds a new IAccountStore to the Application or Organization. The given string can either be an href or a name of a IDirectory or IGroup belonging to the current ITenant.

If the provided value is an href, this method will get the proper Resource and add it as a new AccountStore in this Application or Organization without much effort. However, if the provided value is not an href, it will be considered as a name. In this case, this method will search for both a Directory and a Group whose names equal the provided hrefOrName. If only one resource exists (either a Directory or a Group), then it will be added as a new AccountStore in this Application or Organization. However, if there are two resources (a Directory and a Group) matching that name, a ResourceException will be thrown.

Note: When using names this method is not efficient as it will search for both Directories and Groups within this Tenant for a matching name. In order to do so, some looping takes place at the client side: groups exist within directories, therefore we need to loop through every existing directory in order to find the required Group. In contrast, providing the Group's href is much more efficient as no actual search operation needs to be carried out.

CreateAccountStoreMappingTMapping
Synchronously creates a new IAccountStoreMapping, allowing the associated Account Store to be used a source of accounts that may log in to the Application or Organization.
GetDefaultAccountStoreTMapping
Synchronously gets the IAccountStore (either a IGroup or IDirectory) used to persist new Accounts, or if no default Account Store has been designated.
GetDefaultGroupStoreTMapping
Synchronously gets the IAccountStore used to persist new Groups, or if no default Account Store has been designated.

Stormpath's current REST API requires this to be a IDirectory. However, this could be a IGroup in the future, so do not assume it is always a Directory if you want your code to be function correctly if/when this support is added.

SetDefaultAccountStoreTMapping
Synchronously sets the IAccountStore (either a IGroup or a IDirectory) used to persist new Accounts.

Because this resource is not an IAccountStore itself, it delegates to a Group or Directory when creating accounts; this method sets the IAccountStore to which new account persistence is delegated.

SetDefaultGroupStoreTMapping
Synchronously sets the IAccountStore (a IDirectory) used to persist new Groups.

Stormpath's current REST API requires this to be a IDirectory. However, this could be a IGroup in the future, so do not assume it is always a Directory if you want your code to function properly if/when this support is added.

Because this resource is not an IAccountStore itself, it delegates to a Directory when creating groups; this method sets the IAccountStore to which new group persistence is delegated.
