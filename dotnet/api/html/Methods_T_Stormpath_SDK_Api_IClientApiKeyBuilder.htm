IClientApiKeyBuilder Methods

The IClientApiKeyBuilder type exposes the following members.

Methods
  NameDescription
Build
Constructs a new IClientApiKey instance based on the builder's current configuration state.
SetFileLocation
Sets the location of the .properties file to load containing the API Key ID and Secret used by the Client to communicate with the Stormpath REST API.

When the file is loaded, the following name/value pairs are expected to be present by default: apiKey.id, apiKey.secret

If you want to control the property names used in the file, you may configure them via SetIdPropertyName(String) and SetSecretPropertyName(String).

Note: the tilde (~) character is expanded to %HOMEDRIVE%%HOMEPATH% on Windows, to match the behavior on *nix platforms.

SetId
Allows specifying the client's API Key ID value directly instead of reading it from a file or stream.

Usage Warning: It is almost always advisable to NOT use this method and instead use methods that accept a file or stream: these other methods would ideally acquire the API Key from a secure and private apiKey.properties file that is readable only by the process that uses the Stormpath SDK.

SetIdPropertyName
Sets the name used to represent the API Key ID when parsing a .properties file or stream.
SetInputStream
Creates a IClientApiKey instance from the specified input stream instead of reading from a file. See SetFileLocation(String) for the expected input format.
SetSecret
Allows specifying the client's API Key Secret value directly instead of reading it from a file or stream.

See the security precautions outlined at SetId(String).

SetSecretPropertyName
Sets the name used to represent the API Key Secret when parsing a .properties file or stream.
