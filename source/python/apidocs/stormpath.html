auth Module¶

Authentication library and utilities.

class stormpath.auth.Auth(api_key_file_location=None, api_key_id_property_name='apiKey.id', api_key_secret_property_name='apiKey.secret', api_key=None, id=None, secret=None, scheme='SAuthc1', method=None)¶

Bases: object

Provides authentication for StormPath API requests.

Initialize authentication using one of the available authentication credentials source:

Parameters:
  • api_key_file_location – Location of the API key file (in the Java .properties file format) (default: API key file not used)
  • api_key_id_property_name – Name of the ID property in the key file (default: apiKey.id)
  • api_key_secret_property_name – Name of the secret property in the key file (default: apiKey.secret)
  • api_key – A dictionary-like object containing the id and secret keys mapped to the API ID and secret respectively
  • id – ID (if directly provided; default is None)
  • secret – secret (if directly provided; default is None)
  • scheme – authentication scheme (Sauthc1 or basic, default is Sauthc1)

All available authentication credentials sources are checked, in this priority:

  1. API key file (if api_key_file_location is set and the file exists)
  2. API key dict (if api_key contains id and secret keys)
  3. API key secret and id parameters

The id and secret can be accessed as attributes.

The default authentication scheme is Sauthc1, and is strongly recommended. In cases where it can’t be used (for example, in Google App Engine), basic authentication can be selected instead by supplying basic as the authentication scheme.

Once initialized, scheme property will return the authentication scheme implementation selected.

basic¶

Returns basic http authentication handler which can be used with Python Requests library.

http://docs.stormpath.com/rest/product-guide/#authentication

digest¶

Returns custom authentication handler which can be used with Python requests library, and which uses Stormpath custom digest authentication.

http://docs.stormpath.com/rest/product-guide/#authentication

scheme¶

Selected authentication scheme implementation.

signer¶

Deprecated, use Auth.scheme instead.

class stormpath.auth.Sauthc1Signer(id, secret)¶

Bases: requests.auth.AuthBase

StormPath auth request signer for custom digest auth.

The implementation hooks into the Python Requests library to automatically sign all the requests using this auth mechanism.

More info in documentation: https://www.stormpath.com/docs/rest/api#DigestAuthenticationHTTPS

Initialize new digest auth mechanism with provided id and secret.

client Module¶

Stormpath API client.

class stormpath.client.Client(cache_options=None, expand=None, proxies=None, user_agent=None, **auth_kwargs)¶

Bases: object

The root entry point for SDK functionality.

Using the client instance, you can access all tenant data, such as applications, directories, groups, and accounts.

More info in documentation: http://docs.stormpath.com/python/product-guide/#sdk-concepts

The client contains the following attributes that represent resource lists:

applications - stormpath.resources.application.ApplicationList

directories - stormpath.resources.directory.DirectoryList

accounts - stormpath.resources.account.AccountList

groups - stormpath.resources.group.GroupList

group_memberships - stormpath.resources.group_membership.GroupMembershipList

account_store_mappings - stormpath.resources.account_store_mapping.AccountStoreMappingList

Initialize the client by setting the stormpath.data_store.DataStore and stormpath.resources.tenant.Tenant.

The parameters the Client accepts are those used by the stormpath.http.HttpExecutor, stormpath.auth.Auth, and stormpath.data_store.DataStore classes.

Parameters:user_agent (str) – (optional) The custom user agent to set.

data_store Module¶

Data store abstractions.

class stormpath.data_store.DataStore(executor, cache_options=None)¶

Bases: object

An intermediary between Resource objects and the data they represent.

It fetches the data either from the Stormpath service by using the stormpath.http.HttpExecutor if the cache doesn’t already have it and caches it by using stormpath.cache.manager.CacheManager or fetches the data directly from the cache. Those two components are part of DataStore but implemented separately with the intent of being easily replacable without changing the the rest of the codebase.

Parameters:
  • executor – A HTTP request executor, like stormpath.http.HttpExecutor
  • cache_options – A dictionary with cache settings.

Example of a dictionary with all available options:

{
    'store': MemoryStore,
    'regions': {
        'applications': {
            'store': RedisStore,
            'ttl': 300,
            'tti': 300,
            'store_opts': {
                'host': 'localhost',
                'port': 6739,
            },
        },
        'directories': {
            'store': MemoryStore,
            'ttl': 60,
        },
    },
}

error Module¶

Custom error classes.

exception stormpath.error.Error(error, http_status=None)¶

Bases: exceptions.RuntimeError

Error returned from the StormPath API service.

The string content of the error is the low-level message, intended for the developer. The error also contains the following attributes:

status - The corresponding HTTP status code.

code - A Stormpath-specific error code that can be used to
obtain more information.
developer_message - A clear, plain text explanation with
technical details that might assist a developer calling the Stormpath API.
message - A simple, easy to understand message that you can
show directly to your application end-user.
more_info - A fully qualified URL that may be accessed to
obtain more information about the error.

http Module¶

HTTP request handling utilities.

class stormpath.http.HttpExecutor(base_url, auth, proxies=None, user_agent=None)¶

Bases: object

Handles the actual HTTP requests to the Stormpath service.

It uses the Requests library: http://docs.python-requests.org/en/latest/. The HttpExecutor, along with stormpath.cache.manager.CacheManager is a part of the stormpath.data_store.DataStore.

Parameters:
  • base_url – The root of the Stormpath service. Paths to specific resources will be prepended by this url.
  • auth – Authentication manager, like stormpath.auth.Sauthc1Signer.
