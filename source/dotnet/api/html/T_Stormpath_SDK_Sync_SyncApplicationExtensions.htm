AddAccountStore(IApplication, IAccountStore)
Synchronously adds a new IAccountStore to this Application and appends the resulting IAccountStoreMapping to the end of the Application's AccountStoreMapping list.

If you need to control the order of the added AccountStore, use the CreateAccountStoreMapping(IApplication, IAccountStoreMapping) method.

AddAccountStore(IApplication, String)
Synchronously adds a new IAccountStore to this Application. The given string can either be an href or a name of a IDirectory or IGroup belonging to the current ITenant.

If the provided value is an href, this method will get the proper Resource and add it as a new AccountStore in this Application without much effort. However, if the provided value is not an href, it will be considered as a name. In this case, this method will search for both a Directory and a Group whose names equal the provided hrefOrName. If only one resource exists (either a Directory or a Group), then it will be added as a new AccountStore in this Application. However, if there are two resources (a Directory and a Group) matching that name, a ResourceException will be thrown.

Note: When using names this method is not efficient as it will search for both Directories and Groups within this Tenant for a matching name. In order to do so, some looping takes place at the client side: groups exist within directories, therefore we need to loop through every existing directory in order to find the required Group. In contrast, providing the Group's href is much more efficient as no actual search operation needs to be carried out.

GetDefaultGroupStore
Synchronously gets the IAccountStore used to persist new groups created by the application, or if no account store has been designated.

Stormpath's current REST API requires this to be a Directory. However, this could be a Group in the future, so do not assume it is always a Directory if you want your code to be function correctly if/when this support is added.

ResetPassword
Synchronously verifies the password reset token (received in the user's email) and immediately changes the password in the same request, if the token is valid.

Once the token has been successfully used, it is immediately invalidated and can't be used again. If you need to change the password again, you will previously need to execute SendPasswordResetEmail(IApplication, String) again in order to obtain a new password reset token.

SendPasswordResetEmail(IApplication, String)
Synchronously sends a password reset email for the specified account email address. The email will contain a password reset link that the user can click or copy into their browser address bar.
SendPasswordResetEmail(IApplication, String, IAccountStore)
Synchronously sends a password reset email to an account in the specified IAccountStore matching the specified email. If the email does not match an account in the specified IAccountStore, a ResourceException will be thrown. If you are unsure of which of the application's mapped account stores might contain the account, use the more general SendPasswordResetEmail(IApplication, String) method instead. The email will contain a password reset link that the user can click or copy into their browser address bar.
SendPasswordResetEmail(IApplication, String, String)
Synchronously sends a password reset email to an account matching the specified email in the AccountStore or Organization matching the specified hrefOrNameKey. If the email does not match an account in the specified AccountStore or Organization, a ResourceException will be thrown. If you are unsure of which of the application's mapped account stores might contain the account, use the more general SendPasswordResetEmail(IApplication, String) method instead. The email will contain a password reset link that the user can click or copy into their browser address bar.
SendVerificationEmail(IApplication, ActionEmailVerificationRequestBuilder)
Synchronously triggers the delivery of a new verification email for the specified account.

This method is useful in scenarios where the Account Registration and Verification workflow is enabled. If the welcome email has not been received by a newly registered account, then the user will not be able to login until the account is verified.

This method re-sends the verification email and allows the user to verify the account.

The IEmailVerificationRequest must contain the username or email identifying the account.

SetDefaultAccountStore
Synchronously sets the IAccountStore (either a IGroup or a IDirectory) used to persist new accounts created by the Application.

Because an Application is not an IAccountStore itself, it delegates to a Group or Directory when creating accounts; this method sets the IAccountStore to which the Application delegates new account persistence.

SetDefaultGroupStore
Synchronously sets the IAccountStore (a IDirectory) used to persist new groups created by the Application.

Stormpath's current REST API requires this to be a Directory. However, this could be a Group in the future, so do not assume it is always a Directory if you want your code to function properly if/when this support is added.

Because an Application is not an IAccountStore itself, it delegates to a Directory when creating groups; this method sets the IAccountStore to which the Application delegates new group persistence.
