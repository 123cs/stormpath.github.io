the sendPasswordResetEmail($accountUsernameOrEmail, array $options = array())

Sends a password reset email for the specified account username or email address. The email will contain a password reset link that the user can click or copy into their browser address bar.

This method merely sends the password reset email that contains the link and nothing else. You will need to handle the link requests and then reset the account's password as described in the {@link verifyPasswordResetToken} PHPDoc.

It is possible to include an AccountStore in the $options array as a performance enhancement if the application might be mapped to many (dozens, hundreds or thousands) of account stores. This can be common in multi-tenant applications where each mapped AccountStore represents a specific tenant or customer organization. Specifying the AccountStore in these scenarios bypasses the general email-only-based account search and performs a more-efficient direct lookup directly against the specified AccountStore. The AccountStore is usually discovered before calling this method by inspecting a submitted tenant id or subdomain, e.g. http://ACCOUNT_STORE_NAME.foo.com

Parameters

$accountUsernameOrEmail a username or email address of an Account that may login to the application.
array $options options to pass to this request.

Return Value

the account corresponding to the specified username or email address.

See also

#verifyPasswordResetToken()

the authenticateAccount(AuthenticationRequest $request, array $options = array())

Authenticates an account's submitted principals and credentials (e.g. username and password). The account must be in one of the Application's assigned Login Sources. If not in an assigned login source, the authentication attempt will fail.

Example

Consider the following username/password-based example: 

$request = new UsernamePasswordRequest($email, $submittedRawPlaintextPassword);
$account = $appToTest->authenticateAccount($request)->getAccount();

Parameters

AuthenticationRequest $request the authentication request representing an account's principals and credentials (e.g. username/password) used to verify their identity.
array $options the options to pass to this request.

Return Value

the result of the authentication. The authenticated account can be obtained from the result. {@link AuthenticationResult::getAccount()}.

Exceptions

ResourceError if the authentication attempt fails.
