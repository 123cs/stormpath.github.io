cache Module¶

Cache abstractions.

class stormpath.cache.cache.Cache(store=<class 'stormpath.cache.memory_store.MemoryStore'>, ttl=300, tti=300, **kwargs)¶

Bases: object

A unified interface to different implementations of data caching.

Example of an implementetion is stormpath.cache.memory_store.MemoryStore. It also provides usage statistics with stormpath.cache.stats.CacheStats.

manager Module¶

Cache manager abstraction.

class stormpath.cache.manager.CacheManager¶

Bases: object

Handles all the different caches used by the SDK

It keeps track which resource belongs to which cache. E.g stormpath.resources.directory.Directory resource data is stored in stormpath.cache.memory_store.MemoryStore while stormpath.resources.application.Application resource data is stored in stormpath.cache.redis_store.RedisStore. The CacheManager, along with stormpath.http.HttpExecutor is a part of the stormpath.data_store.DataStore.

redis_store Module¶

A redis cache backend.

class stormpath.cache.redis_store.RedisStore(host='localhost', port=6379, db=0, password=None, socket_timeout=None, connection_pool=None, charset='utf-8', errors='strict', decode_responses=False, unix_socket_path=None, ttl=300)¶

Bases: object

Caching implementation that uses Redis as data storage.

Parameters:
  • host – String representing the hostname or IP of the Redis server
  • port – Port number (int) on which the Redis server is listening
  • db – DB querystring option (Default: 0, e.g. redis://localhost?db=0)
  • password – Redis server password (Default: No Password)
  • socket_timeout – Connection timeout to Redis server
  • connection_pool – ConnectionPool for Redis instance (See redis-py docs)
  • charset – Default character set
  • errors – Default error settings
  • decode_responses – Default: False (values in message dictionaries will be byte strings (str on Python 2, bytes on Python 3)
  • unix_socket_path – Socker path. For using UnixDomainSocketConnection (see redis-py docs for more details)
  • ttl – Default TTL
