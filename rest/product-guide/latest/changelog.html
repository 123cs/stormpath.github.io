Change Log¶

Stormpath is currently on Version 1 of its API, and all APIs use the /v1/ namespace. All changes to the API and documentation are therefore arranged by date, in descending order.

2016-02-24¶

Added section on how to route users to their tenant.

2016-02-19¶

Multi-tenancy chapter significantly expanded. Now includes more information about using Groups vs Directories to model tenants.

2016-02-16¶

  • Added information about IdP-initiated SAML Authentication.
  • Updated Application’s SAML Policy resource in Reference Chapter to include new defaultRelayStates endpoint.

2016-02-10¶

Changed Sphinx Theme to Read The Docs.

2016-02-02¶

  • Updated SAML Section of Authentication chapter to include IdP-specific set-up guides.
  • Added missing organizationAccountStoreMappings resource to Reference chapter.

2016-01-20¶

  • Added Change Log.
  • Directory type language changed. “Social”, “Mirror” and “SAML” Directories are all actually “Mirror Directories”, and the language in the Account Management and Authentication chapters has been updated to reflect this.
  • Created top-level Mirror Directories section that discusses all Mirror Directories in general and how to support multiple Mirror Directories.
  • Updated information about Master Directories and how they function.
